Ez Meeting

Video Conferencing with AI-Powered Summaries & Attention Detection

Project Scope, Flow & Tools Documentation | Version 2.0

1. Project Overview

Ez Meeting is a real-time video conferencing web application with AI-powered meeting summaries, document collaboration, voice captions (English), and participant attention detection using face/eye tracking. Built as a progressive web app (PWA) for instant access on any device.

Instant Meetings

Create or join with a 6-character code or QR scan. No downloads required.

AI Meeting Summary

Automatic post-meeting summary with topics, action items, and key decisions via DeepSeek.

Attention Detection

Real-time face/eye tracking with per-participant active percentage (host-only view).

Document Collaboration

Upload, analyze, and ask AI questions about documents during the meeting.

Voice Captions

Real-time English speech-to-text. Always-on transcript for AI summary.

Meeting History

All participants can view past meetings, summaries, and export reports.

3. Application Flow

3.1 Login Flow

1. User opens app → Login page
↓
2. Enter email + display name → Server issues JWT token
↓
3. Token stored in browser localStorage → Redirected to Lobby

3.2 Meeting Flow

Host: Click "Start Meeting" → Server creates room with 6-char code → Share code/QR/link
↓
Participant: Enter code or scan QR → Join room → WebRTC connections established with all peers
↓
During Meeting: Video/audio streaming, chat, screen share, document upload, voice captions, attention detection all run simultaneously
↓
End Meeting: Host clicks "End for all" OR last person leaves

3.3 Post-Meeting Summary Flow

1. Server compiles transcript (voice captions + chat messages)
↓
2. Transcript sent to DeepSeek API → AI generates structured summary
↓
3. Summary includes: overview, topics, action items, key decisions, participant contributions
↓
4. Meeting saved to Supabase (accessible from "Recent Meetings" by all participants)
↓
5. Users can ask follow-up AI questions, export as text, rename the meeting
↓
6. Attention report shown only to host. Participants see summary without attention data.

3.4 Attention Detection Flow

1. Each user's browser opens a separate low-res camera stream (320x240) for detection
↓
2. MediaPipe FaceLandmarker detects face landmarks every 300ms
↓
3. Per-frame scoring: Eye score (open=1, closed=0) × 0.6 + Head score (forward=1, slight turn=0.5, away=0) × 0.4
↓
4. 3-second sliding window average → Status: Active (>70%), Semi-active (40-70%), Not active (<40%)
↓
5. Score emitted to server every 1s → Server relays to host only → Host sees % on each participant's tile and overall meter

3.5 Document Q&A Flow

1. User uploads document (PDF, Word, Excel, PowerPoint, text, code, etc.)
↓
2. Server extracts text → DeepSeek generates overview, key points, study questions, objectives
↓
3. Insights shown in sidebar → User can ask questions → DeepSeek answers using document context (RAG)

4.3 Attention Detection

ComponentDetail
Detection EngineMediaPipe FaceLandmarker (WASM + GPU), separate 320x240 camera stream
Detection Rate~3 fps (300ms interval)
Eye ScoringEye Aspect Ratio (EAR): open=1, closed=0. Blinks <350ms ignored.
Head ScoringForward=1, slight turn=0.5, looking away/down=0. Grace time: 2s.
WeightingFinal = Eye × 0.6 + Head × 0.4
Smoothing3-second sliding window average
CalibrationFirst 3 seconds: learn user's normal head position as baseline
Status>0.7 = Active, 0.4-0.7 = Semi-active, <0.4 = Not active
VisibilityHost-only. Participants cannot see any attention data. Host's own % excluded from overall meter.
Emit RateScore sent to server every 1 second
